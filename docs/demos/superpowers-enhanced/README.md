# Superpowers Enhanced Demo Index

Use this folder for screenshots, short GIFs, terminal snippets, and Markdown excerpts that show Superpowers Enhanced operating in real agent sessions.

## Demo Checklist

Each demo should include:

- **Skill:** one of `brainstorming-enhanced`, `writing-plans`, `subagent-driven-development`, or `executing-plans`.
- **Client tested:** the agent client used for the demo.
- **User request:** the short prompt or scenario that started the workflow.
- **What changed:** the quality-of-life improvement the skill provided.
- **Artifact:** screenshot, GIF, terminal snippet, spec excerpt, plan excerpt, or review excerpt.
- **Result:** the concrete outcome.

## Recommended Files

| File | Purpose |
| --- | --- |
| `brainstorming-enhanced-browser.md` | Show browser-first questions, approach choices, and design approval screens. |
| `writing-plans-example.md` | Show how a spec becomes a detailed implementation plan. |
| `subagent-review-loop.md` | Show spec and quality review loops catching issues during implementation. |
| `executing-plans-inline.md` | Show inline execution for clients without subagent support. |

## Asset Folders

Put larger media next to the demo Markdown:

```text
brainstorming-enhanced-browser/
  approval-screen.png
  approach-selection.gif
```

Keep screenshots focused on the decision or artifact being demonstrated. Avoid full desktop captures unless the surrounding UI is important.
